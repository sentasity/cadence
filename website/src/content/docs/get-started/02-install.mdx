With the mental model in hand — six stages, readable artifacts, gates you approve — installing is quick. The plugin installs once and then works in every repository on your machine.

## Install the plugin

<Tabs>
  <TabItem label="Install commands">
```text
/plugin marketplace add sentasity/cadence
/plugin install cadence@cadence
```
  </TabItem>
  <TabItem label="What you'll see">
After a session restart, all nine `/c-*` commands are available. There's no separate setup step — the first time you run `/c-brainstorm` in a fresh repo, Cadence offers to scaffold its config for you (you'll do that in the next lesson).
  </TabItem>
</Tabs>

<Aside type="caution" title="Restart your session">
The commands appear only after you restart your Claude Code session. If `/c-brainstorm` isn't recognized, restart and try again.
</Aside>

## Confirm the nine commands are there

<Walkthrough title="Confirm Cadence is installed">
  <Fragment slot="step-1-title">Restart your Claude Code session</Fragment>
  <Fragment slot="step-1-body">After `/plugin install cadence@cadence`, fully restart the session so the plugin's commands register.</Fragment>
  <Fragment slot="step-2-title">Type a slash and look for `/c-`</Fragment>
  <Fragment slot="step-2-body">Begin typing `/c-` in the prompt. You should see the six core commands (`/c-brainstorm`, `/c-design`, `/c-plan`, `/c-execute`, `/c-audit`, `/c-validate`) and the three diagnostics (`/c-check`, `/c-find-bugs`, `/c-explain`).</Fragment>
  <Fragment slot="step-3-title">You're ready</Fragment>
  <Fragment slot="step-3-body">Nine commands visible means the plugin is live in this repo and every other repo on your machine. Move on to your first brainstorm.</Fragment>
</Walkthrough>

Throughout the rest of Get Started you'll build one tiny feature — a function that prints the current time — and at the end compare your output against the finished worked example shipped with Cadence at `examples/hello-cadence/`. Same shape of artifacts means you did it right.

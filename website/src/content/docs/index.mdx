Cadence is a set of Claude Code skills that move a software idea through a repeatable loop: **brainstorm → design → plan → execute → audit → validate**. Each stage produces a reviewable artifact, so work stays legible and nothing is silently skipped.

New here? Walk the [Get Started guide](/cadence/get-started/01-what-is-cadence/) to run one full loop end to end. Already know the flow? Jump straight to the [command reference](/cadence/reference/c-brainstorm/).
